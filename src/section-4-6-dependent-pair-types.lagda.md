# Section 4.6 Dependent pair types

```agda
module section-4-6-dependent-pair-types where

open import universe-levels
```

Given a type family `B` over `A`, we may consider pairs `(a , b)` of terms, where `a : A` and `b : B(a)`.
Note that the type of `b` depends on the first term in the pair.
Therefore we call such a pair a **dependent pair**.
The type of such dependent pairs is the inductive type that is generated by the dependent pairs.

## Definition 4.6.1

Consider a type family `B` over `A`.
The **dependent pair type** (or **`Σ`-type**) is defined to be the inductive type `Σ(x : A) B(x)` equipped with a **pairing function**

```text
  pair : Π(x : A) (B(x) → Σ(y : A) B(y)).
```

The induction principle for `Σ(x : A) B(x)` asserts that for any family of types `P(p)` indexed by `p : Σ(x : A) B(x)`, there is a function

```text
  ind-Σ :
    (Π(x : A) Π(y : B(x)) P(pair(x , y))) →
    (Π(z : Σ(x : A) B(x)) P(z)).
```

satisfying the computation rule

```text
  ind-Σ(g , pair(x , y)) ≐ g(x , y).
```

Alternatively, a definition of a dependent function `f : Π(z : Σ(x : A) B(x)) P(z)` by induction using a function `g : Π(x : A) Π(y : B(x)) P((x , y))` can be presented by pattern matching as

```text
  f(pair(x , y)) := g(x , y).
```

We will usually write `(x , y)` for `pair(x , y)`.

```agda
record Σ {l1 l2 : Level} (A : Type l1) (B : A → Type l2) :
  Type (l1 ⊔ l2)
  where
  constructor pair
  field
    pr1 : A
    pr2 : B pr1

open Σ public

{-# INLINE pair #-}

infixr 3 _,_
pattern _,_ a b = pair a b

ind-Σ :
  {l1 l2 l3 : Level} {A : Type l1} {B : A → Type l2}
  {C : Σ A B → Type l3} →
  ((x : A) (y : B x) → C (x , y)) → (t : Σ A B) → C t
ind-Σ f (x , y) = f x y

rec-Σ :
  {l1 l2 l3 : Level} {A : Type l1} {B : A → Type l2}
  {C : Type l3} →
  ((x : A) → B x → C) → Σ A B → C
rec-Σ = ind-Σ
```

The induction principle of `Σ`-types can be used to define the projection functions.

## Definition 4.6.2

Consider a type `A` and a type family `B` over `A`.

1. The **first projection map**

```text
  pr1 : (Σ(x : A) B(x)) → A
```

is defined by induction as

```text
  pr1(x , y) := x.
```

2. The **second projection map** is a dependent function

```text
  pr2 : Π(p : Σ(x : A) B(x)) B(pr1(p)),
```

defined by induction as

```text
  pr2(x , y) := y.
```

### Remark 4.6.3

If we want to construct a function

```text
  f : Π(z : Σ(x : A) B(x)) P(z)
```

by `Σ`-induction, then we get to assume a pair `(x , y)` consisting of `x : A` and `y : B(x)` and our goal will be to construct an element of type `P(x , y)`.
The induction principle of `Σ`-types is therefore converse to the **currying operation**, a familiar concept from the theory of programming languages, which is given by the function

```text
  ev-pair :
    (Π(z : Σ(x : A) B(x)) P(z)) →
    (Π(x : A) Π(y : B(x)) P(x , y))
```

given by `f ↦ λ x. λ y. f(x , y)`.
The induction principle `ind-Σ` is therefore also known as the **uncurrying operation**.

```agda
ev-pair :
  {l1 l2 l3 : Level} {A : Type l1} {B : A → Type l2}
  {C : Σ A B → Type l3} →
  ((t : Σ A B) → C t) → (x : A) (y : B x) → C (x , y)
ev-pair f x y = f (x , y)
```

A common special case of the `Σ`-type occurs when the `B` is a constant family over `A`, i.e., when `B` is just a type weakened by `A`.
In this case, the type `Σ(x : A) B` is the type of *ordinary* pairs `(x , y)` where `x : A` and `y : B`, where the type of `y` does not depend on `x`.
The type of ordinary pairs `(x , y)` consisting of `x : A` and `y : B` is of course the *product* of `A` and `B`, so we see that product types arise as a special case of `Σ`-types in a similar way to how function types were defined as a special case of `Π`-types.

## Definition 4.6.4

Consider two types `A` and `B`.
Then we define the **(cartesian) product** `A × B` of `A` and `B` by

```text
  A × B := Σ(x : A) B.
```

```agda
product : {l1 l2 : Level} (A : Type l1) (B : Type l2) → Type (l1 ⊔ l2)
product A B = Σ A (λ _ → B)

pair' :
  {l1 l2 : Level} {A : Type l1} {B : Type l2} →
  A → B → product A B
pair' = pair

infixr 15 _×_
_×_ : {l1 l2 : Level} (A : Type l1) (B : Type l2) → Type (l1 ⊔ l2)
_×_ = product
```

### Remark 4.6.5

Since `A × B` is defined as a `Σ`-type, it follows that cartesian products also satisfy the induction principle of `Σ`-types.
In this special case, the induction principle for `A × B` asserts that for any type family `P` over `A × B` there is a function

```text
  ind-× :
    (Π(x : A) Π(y : B) P(x , y)) →
    (Π(z : A × B) P(z))
```

that satisfies the computation rule

```text
  ind-×(g , (x , y)) ≐ g(x , y).
```

```agda
ind-product :
  {l1 l2 l3 : Level} {A : Type l1} {B : Type l2}
  {C : A × B → Type l3} →
  ((x : A) (y : B) → C (x , y)) → (t : A × B) → C t
ind-product = ind-Σ

rec-product :
  {l1 l2 l3 : Level} {A : Type l1} {B : Type l2} {C : Type l3} →
  (A → B → C) → A × B → C
rec-product = ind-product
```

The projection maps are defined similarly to the projection maps of `Σ`-types.
When one thinks of types as propositions, then `A × B` is interpreted as the conjunction of `A` and `B`.
